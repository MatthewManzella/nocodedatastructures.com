Deque





Definition: A collection of elements that are stored next to each other in a line. Elements can only be inserted and deleted from the ends of the structure. Short for double ended queue.



Real Life Example: Think of an NFL game. Before the ball is snapped, linemen from both teams are huddled right by the ball. Players on each end are standing behind the linemen and even more players are standing behind the second line. When the ball is snapped and everyone jumps towards each other, a deque is created. Players can join or leave the pile from the very outside but can't just show up in the middle.


Note: The first element in a deque (farthest left) is referred to as the "front". The last element in a deque (farthest right) is referred to as the "rear" or "back".



Basic Deque Operations:

  1. Pushing an element onto the front or rear. An element is added to the deque at the specified end and becomes the front/rear.
  2. Popping an element from the front or rear. The outermost element at the specified side is removed from the deque and the closest element in line becomes the front/rear.


Fun Fact: Deque is pronounced "deck" not "DQ".



Commonly Used In:

  • Data stream processing tasks.
  • Implementing advanced graph algorithms.
  • Implementing other data structures.


Other Similar Data Structures:

  • Linked Deque: A deque in which elements are linked to their neighbors via a doubly linked list.

  • Block Deque: A deque that stores collections of elements at each slot rather than individual elements at for the sake of efficiency.

  • Circular Deque: A deque in which elements are organized in a circle rather than a line so that less memory is wasted at the expense of constant front/rear insertions and deletions.
